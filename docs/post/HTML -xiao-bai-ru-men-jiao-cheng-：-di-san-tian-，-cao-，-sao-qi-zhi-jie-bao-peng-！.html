复习前两天：你他妈已经很牛逼了

先来把前两天的东西过一遍，复习提示帮你把基础焊死在脑子里。忘了？操，忘了也得记，像背你家 Wi-Fi 密码！

第一天：基础标签，简单得像吃薯片

  • HTML 是啥：网页的骨架，像搭积木，告诉浏览器怎么显示内容。
    • 复习提示：HTML 就像房子的地基，CSS 是油漆，JavaScript 是电灯。HTML 只管结构，不管样子。
  • 结构：每个 HTML 文件的固定套路，像写信的格式：
    • <!DOCTYPE html>：告诉浏览器这是 HTML5，写在最开头。
    • <html>：整个网页的容器，包住所有内容。
    • <head>：放元信息（标题、编码），不显示在页面。
    • <body>：放网页内容（文字、图片），显示在页面。
    • 复习提示：<head> 像书的目录，<body> 像书的正文。忘了 <!DOCTYPE html>，浏览器可能抽风。
  • 核心标签：
    • <h1>-<h6>：标题，从大到小，像报纸头条。<h1> 最大，<h6> 最小。
      • 复习提示：<h1> 是大老板，<h6> 是小弟，用来分级标题，别乱用。
    • <p>：段落，写大段文字，自动换行，像日记本。
      • 复习提示：<p> 是讲故事的，每次用都自带换行，别跟 <br> 搞混。
    • <a>：链接，点一下跳到别的网页，像传送门。
      • 复习提示：<a href="https://www.bilibili.com">B 站</a> 就像超链接，href 是目的地，忘了 https:// 就点不动。
    • <img>：图片，展示猫狗图，像贴海报。
      • 复习提示：<img src="dog.jpg" alt="狗"> 的 src 是图片地址，alt 是图片描述，图片挂了显示 alt。
    • <br>：换行，强制断行，像按 Enter。
      • 复习提示：<br> 是单身狗标签，不用闭合，直接插在想换行的地方。
  • 成果：你写了带标题、段落、链接、图片的网页，打开浏览器，爽得像吃火锅。
  • 踩过的坑：
    • 乱码：忘了 <meta charset="UTF-8">，中文变方块。
      • 复习提示：<meta charset="UTF-8"> 放 <head> 里，像告诉浏览器“哥们，用中文编码”。
    • 图片不显示：路径错（Dog.jpg ≠ dog.jpg）。
      • 复习提示：图片路径像找文件，写错地址（大小写、文件夹）就找不到。
    • 链接点不动：href 写错（漏 https://）。
      • 复习提示：href 是链接的 GPS，写全地址（https://www.google.com），别偷懒。

第二天：列表、表格、表单，骚气初显

  • 列表：
    • <ul>：无序列表，带圆点，像购物清单。
      • 复习提示：<ul> 是“unordered list”，每项前面是圆点，适合列没顺序的东西（比如爱好）。
    • <ol>：有序列表，带数字，像做菜步骤。
      • 复习提示：<ol> 是“ordered list”，自动编号，适合列步骤（比如 1、2、3）。
    • <li>：列表项，放具体内容，嵌在 <ul> 或 <ol> 里。
      • 复习提示：<li> 是列表的每一行，忘了放 <ul> 或 <ol> 里，浏览器会懵。
  • 表格：
    • <table>：表格容器，像 Excel。
      • 复习提示：<table> 是整个表格的框，里面放行和格子。
    • <tr>：一行（table row），像 Excel 的行。
      • 复习提示：<tr> 是“table row”，每行用一个 <tr>。
    • <th>：表头（table header），加粗居中，像标题。
      • 复习提示：<th> 是表格的“大佬格”，默认粗体，写在第一行。
    • <td>：普通格子（table data），放数据。
      • 复习提示：<td> 是“table data”，普通格子，每行数量要对齐。
  • 表单：
    • <form>：表单容器，收用户输入，像问卷。
      • 复习提示：<form> 是收集信息的框，里面放输入框、按钮。
    • <input>：输入框，收文字、密码，像填表。
      • 复习提示：<input type="text"> 是普通输入，type="password" 是密码框，placeholder 是提示文字。
    • <button>：按钮，点一下提交，像发送键。
      • 复习提示：<button> 默认是提交，点它没反应正常，后面学 JavaScript 加功能。
  • CSS 基础：
    • 内联 style（style="color: red;"），改颜色、字体、边框。
      • 复习提示：style 是给标签化妆，写在标签里，比如 <p style="color: blue;"> 让文字变蓝。
  • 成果：网页有了列表、表格、表单，背景浅灰，标题蓝色，按钮红底白字，骚得像夜店。
  • 踩过的坑：
    • 列表乱：<li> 没放 <ul> 或 <ol>。
      • 复习提示：<li> 必须有“爹”（<ul> 或 <ol>），单干会乱。
    • 表格歪：每行 <td> 数量不一致。
      • 复习提示：表格像 Excel，每行格子数要一样，不然歪得像斜塔。
    • CSS 失效：忘了分号（;）或值错（color: redd）。
      • 复习提示：CSS 每条属性要加 ;，值得写对（red 不是 redd）。

今天我们加点新料，复习完是不是感觉像吃了颗定心丸？操，继续冲！

新技能 1：规范标签，网页像穿了西装

以前的标签（比如 <div>）像万能胶，啥都能干，但代码乱得像杂货铺。现在学点新标签，让网页结构清晰得像书本目录。你以为写网页只是给浏览器看？操，搜索引擎和盲人用户也得伺候好！

新标签：让代码更有逼格

  • <header>：网页顶部，放标题或导航，像书的封面。
    • 复习提示：<header> 像网页的“抬头”，放 <h1> 或 <nav>，别塞乱七八糟的东西。
  • <nav>：导航栏，放链接，像网站菜单。
    • 复习提示：<nav> 是“navigation”，放 <a> 链接，点一下跳页面。
  • <main>：主要内容，放核心东西，像书的核心章节。
    • 复习提示：<main> 是网页的“正文”，放主要内容，别跟 <header> 或 <footer> 混。
  • <footer>：网页底部，放版权、联系方式，像书的封底。
    • 复习提示：<footer> 是“尾巴”，放 © 信息或联系方式。

示例：

<header>
    <h1>我的牛逼网站</h1>
</header>
<nav>
    <a href="#home">首页</a> | <a href="#about">关于</a>
</nav>
<main>
    <h2>欢迎</h2>
    <p>这他妈是第三天，网页骚气拉满！</p>
</main>
<footer>
    <p>© 2025 老子牛逼</p>
</footer>

效果（没 CSS，纯结构）：

  • 顶部：大标题“我的牛逼网站”。
  • 导航：链接“首页 | 关于”。
  • 主体：欢迎标题和段落。
  • 底部：版权信息。

为啥用这些标签：

  • 清晰：代码像菜单，结构一目了然，找东西快。
  • 搜索引擎爱：Google、Baidu 更容易懂，排名可能高。
  • 盲人友好：读屏软件认得出 <header>、<nav>，盲人用户点赞。
  • 装逼：2025 年，这些标签是前端标配，面试官看了点头。

坑提醒：

  • 别乱用，<header> 只放标题或导航，别塞 <p>，否则像西装里塞棉袄。
  • 标签没闭合（<header>xxx），结构乱，用 VS Code 看缩进。
  • 这些标签没样式，丑得像白馒头，后面用 CSS 化妆。
  • <div> 还是万能胶，辅助用，别全换成新标签。

新技能 2：CSS 单独文件，样式整洁像五星酒店

前两天用内联 CSS（style="color: red;"），像在网页上刷油漆，改一处还行，改多了乱得像猪窝。单独 CSS 文件像把样式装进行李箱，统一管理，改起来爽得像吃冰淇淋。你以为 CSS 是给网页上色？操，这是给网页整容！

怎么搞单独 CSS

  1. 建 CSS 文件：

    • 在 my_webpage_day3 文件夹建 style.css。

    • 写样式，比如：

      h1 {
    color: blue; /* 标题蓝色 */
    font-size: 30px; /* 字体 30 像素 */
    text-align: center; /* 居中 */
}
p {
    color: green; /* 段落绿色 */
    font-size: 18px; /* 字体 18 像素 */
}

  2. 连到 HTML：

    • 在 HTML 的 <head> 加 <link>：

      <head>
    <meta charset="UTF-8">
    <title>第三天，操，骚气爆棚！</title>
    <link rel="stylesheet" href="style.css">
</head>

  3. 效果：所有 <h1> 变蓝色、30px、居中，<p> 变绿色、18px。

CSS 术语（中文）：

  • 选择器：指明改谁，比如 h1（改所有标题）、p（改所有段落）。
    • 复习提示：选择器像“点名”，h1 点所有 <h1>，错了（H1）没人理。
  • 属性：改啥，比如 color（颜色）、font-size（字体大小）。
    • 复习提示：属性是“化妆工具”，color 是染发剂，font-size 是放大镜。
  • 值：改成啥，比如 blue（蓝色）、18px（18 像素）。
    • 复习提示：值是“化妆效果”，blue 是蓝色，18px 是大小，写错（blu）无效。
  • 格式：选择器 { 属性: 值; 属性: 值; }，每条属性用分号 ; 隔开。
    • 复习提示：分号像句号，忘了 ; 后面样式可能罢工。

常用 CSS 属性（中文）：

  • color：文字颜色（red 红，#FF0000 十六进制红，rgb(255, 0, 0) RGB 红）。
  • font-size：字体大小（18px 像素，1.5em 相对大小）。
  • background-color：背景颜色（#f0f0f0 浅灰）。
  • text-align：文字对齐（center 居中，left 左，right 右）。
  • width：宽度（200px 像素，50% 百分比）。
  • margin：外边距（20px 外面空 20 像素）。
  • padding：内边距（10px 内容与边框间空 10 像素）。

坑提醒：

  • CSS 文件连不上：
    • 确认 <link href="style.css">，style.css 跟 HTML 同文件夹。
    • 文件名大小写敏感（Style.css ≠ style.css）。
    • 路径错（href="css/style.css" 但 CSS 不在 css 文件夹），用 Chrome F12 “Network”看 CSS 是否加载.
    • 复习提示：<link> 是 CSS 的“桥梁”，路径错了桥就塌。
  • CSS 不生效：
    • 忘了分号（color: blue font-size: 18px 错，改 color: blue; font-size: 18px;）。
    • 选择器错（H1 无效，h1 正确）。
    • 值错（color: redd 无效，color: red 正确）。
    • 复习提示：CSS 像写信，格式不对（漏 ;）、地址错（选择器）、内容错（值），信送不到。
  • 缓存问题：
    • 改了 CSS 没效果？浏览器缓存，按 Ctrl + F5 强制刷新。
    • Live Server 不更新？确认保存 HTML 和 CSS.
    • 复习提示：缓存像旧照片，强制刷新（Ctrl + F5）换新照。
  • VS Code 提示：
    • 装 CSS Peek 插件，鼠标点选择器跳到定义。
    • 用 Auto Rename Tag 插件，改标签自动同步闭合。
    • 复习提示：VS Code 插件像“助手”，帮你少犯错。

新技能 3：简单导航，网页像有了方向盘

网页得有导航条，像餐厅菜单，点一下跳到对应区域。你以为导航只是加几个链接？操，这是给网页装 GPS，点哪到哪！

用 <nav> 做导航

  • <nav> 放链接，配合 <a>。

  • 示例：

    <nav>
    <a href="#home">首页</a> | 
    <a href="#about">关于</a> | 
    <a href="https://www.bilibili.com" target="_blank">B 站</a>
</nav>

  • 效果：显示“首页 | 关于 | B 站”，点“B 站”跳到 B 站，其他是占位符。

CSS 美化导航：

  • 让链接横排、去下划线、加颜色。

  • 示例 CSS：

    nav a {
    color: white; /* 链接白色 */
    text-decoration: none; /* 去下划线 */
    margin: 0 10px; /* 链接间空 10 像素 */
}
nav {
    background-color: blue; /* 导航蓝底 */
    padding: 10px; /* 内边距 10 像素 */
    text-align: center; /* 居中 */
}

  • 效果：导航蓝底，链接白色，无下划线，居中，间距舒服。

坑提醒：

  • 链接没反应？检查 href（https:// 不能少，#home 是占位符）。
    • 复习提示：<a> 的 href 像导航仪，地址错了（漏 https://）到不了。
  • CSS 不生效？确认选择器（nav a 针对 <nav> 里的 <a>）。
    • 复习提示：选择器像“点名”，nav a 只点 <nav> 里的 <a>。
  • 导航太挤？加 margin 或 padding 拉开距离.
  • Want hover effect (color change on mouseover)? Learn :hover later.

实战：写个简单骚气的网页

整合今天学的，写个用新标签、单独 CSS、导航的网页。假设有图片 dog.jpg，CSS 文件 style.css。

文件结构

my_webpage_day3/
├── index.html
├── style.css
├── dog.jpg

HTML 代码（index.html）

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>第三天，操，骚气冲天！</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- 顶部 -->
    <header>
        <h1>欢迎来我的牛逼网站</h1>
        <nav>
            <a href="#home">首页</a> | 
            <a href="#about">关于</a> | 
            <a href="https://www.bilibili.com" target="_blank">B 站</a>
        </nav>
    </header>
    
    <!-- 主体 -->
    <main>
        <h2>欢迎，兄弟！</h2>
        <p>这是我第三天学的网页，操，简单又骚气！</p>
        <img src="dog.jpg" alt="一只帅狗">
        <h2>我的计划</h2>
        <ul>
            <li>学 HTML，搞定基础</li>
            <li>学 CSS，网页变漂亮</li>
            <li>学 JavaScript，网页会动</li>
        </ul>
    </main>
    
    <!-- 底部 -->
    <footer>
        <p>© 2025 老子牛逼</p>
    </footer>
</body>
</html>

CSS 代码（style.css）

/* 整体 */
body {
    background-color: #f0f0f0; /* 背景浅灰 */
    font-family: Arial; /* 字体用 Arial */
    margin: 0; /* 去掉默认外边距 */
}

/* 顶部 */
header {
    background-color: #007BFF; /* 蓝底 */
    color: white; /* 文字白 */
    text-align: center; /* 居中 */
    padding: 15px; /* 内边距 15 像素 */
}

header h1 {
    font-size: 32px; /* 标题 32 像素 */
    margin: 0; /* 去掉默认外边距 */
}

/* 导航 */
nav a {
    color: white; /* 链接白 */
    text-decoration: none; /* 去下划线 */
    margin: 0 10px; /* 链接间空 10 像素 */
}

/* 主体 */
main {
    margin: 20px; /* 外边距 20 像素 */
}

main h2 {
    color: #28A745; /* 绿色 */
    font-size: 24px; /* 24 像素 */
}

main p {
    font-size: 18px; /* 18 像素 */
    color: #333; /* 深灰 */
}

main img {
    width: 300px; /* 图片宽 300 像素 */
    margin: 10px 0; /* 上下空 10 像素 */
}

main ul {
    font-size: 16px; /* 列表 16 像素 */
}

/* 底部 */
footer {
    background-color: #333; /* 黑底 */
    color: white; /* 文字白 */
    text-align: center; /* 居中 */
    padding: 10px; /* 内边距 10 像素 */
}

怎么运行

  1. 在 my_webpage_day3 文件夹建 index.html 和 style.css，复制代码。
  2. 放 dog.jpg 到文件夹（没图片去 unsplash.com 下，改名）。
  3. 用 VS Code 打开，装 Live Server，点“Go Live”预览，或拖 index.html 到 Chrome。
  4. 保存文件（Ctrl + S），刷新浏览器（Ctrl + F5），看效果。

效果

网页像个简洁博客，骚气又不乱：

  • 标题：第三天，操，骚气冲天！
  • 整体：
    • 背景浅灰，内容区域 20px 外边距。
    • 顶部：蓝底白字，标题居中，导航链接（首页、关于、B 站）白色，间距舒服。
    • 主体：绿色副标题，18px 段落，300px 狗图，列表列计划。
    • 底部：黑底白字，居中版权。
  • 视觉：蓝绿灰黑搭配，字体清晰，图片大小合适，像小网站雏形。

坑提醒：

  • CSS 不生效：
    • 确认 <link href="style.css"> 路径，style.css 跟 HTML 同目录。
    • CSS 语法错（漏 ; 或选择器错），用 Chrome F12 “Styles”查。
    • 缓存问题，Ctrl + F5 刷新.
    • 复习提示：CSS 不生效像没插电源，检查路径、语法、缓存。
  • 图片不显示：
    • 确认 dog.jpg 在文件夹，大小写一致。
    • 压缩到 1MB（tinypng.com）。
    • 路径用 dog.jpg，别用 C:\XXX\dog.jpg.
    • 复习提示：图片不显示像挂错画，检查 src 和文件位置。
  • 导航没效果：
    • href="#home" 是占位符，B 站链接要完整（https://www.bilibili.com）。
    • CSS 选择器错（nav a 不是 nav）。
    • 复习提示：导航像路标，href 错了路就断。
  • 结构乱：
    • 检查 <header>、<main>、<footer> 是否闭合，用 VS Code 缩进看。
    • 标签嵌套错（<header><p>xxx</header></p>），用 Chrome F12 “Elements”查。
    • 复习提示：结构乱像搭错积木，检查闭合和嵌套。

常见问题：别他妈慌，坑都给你踩过了

前两天的坑可能还遇到，今天加新坑，快速解决：

  1. 新标签没效果？

    • 原因：<header>、<nav> 只管结构，没样式，丑正常。
    • 解决：
      • 在 style.css 加样式（header { background-color: blue; }）。
      • 确认闭合（<header>xxx</header>）。
      • 用 Chrome F12 “Elements”看结构.
      • 复习提示：新标签像白纸，CSS 是颜料，不涂就丑。

  2. CSS 文件没加载？

    • 原因：路径错、文件没保存、语法错。
    • 解决：
      • 确认 <link href="style.css">，style.css 在文件夹。
      • 检查文件名（Style.css ≠ style.css）。
      • 保存 CSS 和 HTML（Ctrl + S）。
      • 用 Chrome F12 “Network”，看 CSS 是否加载，红叉说明路径错.
      • Revision Tip: <link> is the CSS lifeline; wrong path, no styles.

  3. Styles Not Working?

    • 原因：选择器错、值错、忘了分号。
    • 解决：
      • 选择器用 h1、p、nav a，别写 H1 或 nav a a。
      • 值正确（color: red 不是 color: redd）。
      • 每条属性加 ;（color: blue; font-size: 18px;）。
      • 用 Chrome F12 “Styles”查.
      • 复习提示：CSS 像菜谱，材料（值）、步骤（分号）错，菜就没法吃。

  4. 导航链接太挤？

    • 原因：没加间距。
    • 解决：
      • 加 CSS（nav a { margin: 0 10px; }）。
      • 确认选择器（nav a 针对 <nav> 里的 <a>）。
      • 复习提示：导航挤像人挤地铁，加 margin 给空间。

  5. 网页在手机上乱？

    • 原因：没适配移动端。
    • 解决：
      • 今天用 width: 300px 固定图片，后面学响应式。
      • 用 Chrome F12 “设备”图标，模拟手机看效果.
      • 复习提示：手机显示乱像衣服不合身，后面学“裁缝”技巧。

  6. VS Code 或 Live Server 抽风？

    • 原因：插件问题、端口占、没保存。
    • 解决：
      • 确认 Live Server 启用（底部“Go Live”）。
      • 端口 5500 被占，改 5501（VS Code 设置）。
      • 保存所有文件（Ctrl + S）。
      • VS Code 卡？关无关扩展，重启.
      • 复习提示：Live Server 像“实时监控”，不保存就看不到最新。

  • <img>：图片，像海报，src 是路径。
  • <ul>、<ol>、<li>：列表，像清单，<li> 必须有爹。
  • <table>、<tr>、<th>、<td>：表格，像 Excel，格子要对齐。
  • <form>、<input>、<button>：表单，像问卷，收用户输入。
