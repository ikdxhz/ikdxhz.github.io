## 啥是 HTML？别慌，简单说两句

HTML，全称 **HyperText Markup Language**，中文叫“超文本标记语言”。听起来有点装逼，但其实就是一种用来描述网页结构的语言。你可以把它看成房子的骨架，CSS 是刷墙搞装修，JavaScript 是装空调加智能家居。HTML 负责把网页的内容（文字、图片、链接、视频）组织好，告诉浏览器“嘿，这是个标题，那是段落，旁边放张图”。

- **为啥学 HTML？** 因为这是做网页的第一步，学会了 HTML，你就能搞自己的网站，装逼、赚钱、搞副业，随你挑。操，现在连摆摊的大妈都知道做个网页能多卖点货，你还不学？
- **难不难？** 难个屁！HTML 就是一堆标签（tag），你学会怎么写标签，基本就入门了。比玩消消乐还简单。

**今天的目标**：搞懂 HTML 基础，写一个包含标题、段落、图片、链接的简单网页，还要知道哪里容易出错，咋解决。我们不整复杂的东西，慢慢来，保你爽得飞起。

## 准备工作：别磨叽，赶紧干

要写 HTML，你不需要什么高大上的装备，电脑上有个能打字的软件就行。以下是你需要的东西，简单列一下：

1. **文本编辑器**：

## HTML 基本结构：骨架是啥样？

HTML 文件就是一堆文本，后缀是 `.html`（比如 `index.html`）。每个 HTML 文件都有个固定的骨架，记住这个结构，你就成功一半了。

下面是最简单的 HTML 骨架，直接抄，后面解释：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>我的第一个网页，牛逼炸了！</title>
</head>
<body>
    <h1>欢迎来学 HTML，兄弟！</h1>
    <p>这他妈是我写的第一个网页，爽爆了！</p>
</body>
</html>
```

### 逐行拆解，搞清楚每部分干啥

- `<!DOCTYPE html>`：
  - 这行告诉浏览器，“老子用的是 HTML5 标准，别搞乱”。必须写在第一行，固定格式，背下来。
  - **坑提醒**：
    - 漏写这行，浏览器可能用老标准渲染，页面排版会乱。
    - 写错（比如大小写错误），有些浏览器会忽略，但别冒险。

- `<html lang="zh-CN">`：
  - `<html>` 是整个网页的根标签，所有内容都得包在这对标签里。
  - `lang="zh-CN"` 表示网页用中文，方便搜索引擎和读屏软件。英文网页用 `lang="en"`。
  - **坑提醒**：
    - 忘了闭合 `</html>`，浏览器可能照样显示，但代码不规范，容易出问题。
    - `lang` 属性写错（比如 `zh-cn` 小写），影响不大，但规范点好。

- `<head>`：
  - 头部标签，里面放“幕后信息”，比如网页标题、编码格式。
  - `<meta charset="UTF-8">`：告诉浏览器用 UTF-8 编码，中文不会乱码。固定写法，抄就行。
  - `<title>`：网页标题，显示在浏览器标签栏。比如 `<title>我的第一个网页，牛逼炸了！</title>`。
  - **坑提醒**：
    - 漏写 `<meta charset="UTF-8">`，中文可能变乱码（一堆 `???` 或方块）。
    - `<title>` 忘了写，浏览器标签栏会显示文件名（比如 `index.html`），low 爆了。
    - `<head>` 没闭合 `</head>`，浏览器可能照样跑，但代码不规范。

- `<body>`：
  - 网页的“身体”，所有你能在网页上看到的内容（文字、图片、视频）都放这儿。
  - `<h1>` 是大标题，`<p>` 是段落，后面细讲。
  - **坑提醒**：
    - `<body>` 没闭合，内容可能显示不全。
    - 标签嵌套错误（比如 `<h1><p>xxx</h1></p>`），浏览器会尝试修复，但效果可能乱。

## HTML 标签：核心玩法，学几个就够

HTML 的精髓是**标签**（tag）。标签就是用 `<` 和 `>` 包起来的关键字，比如 `<h1>`、`<p>`。大多数标签成对出现，有开头和结尾（比如 `<h1>内容</h1>`）。也有少数单身狗标签（比如 `<img>`），不需要闭合。

### 标签的规则

- 标签不区分大小写，但建议小写（比如 `<h1>` 比 `<H1>` 规范）。
- 标签可以嵌套，但不能交叉（比如 `<h1><p>xxx</p></h1>` 合法，`<h1><p>xxx</h1></p>` 会乱）。
- 属性写在开头标签里，格式是 `属性名="值"`，值用引号包起来（单引号双引号都行）。

### 常用标签：第一天这几个就够装逼

今天只学几个最常用的标签，简单但实用：

1. **标题标签：`<h1>` 到 `<h6>`**
   - 用来写标题，`<h1>` 最大，`<h6>` 最小。
   - 示例：
     ```html
     <h1>超大标题，牛逼！</h1>
     <h2>稍小点的标题</h2>
     <h3>更小</h3>
     <h4>再小</h4>
     <h5>很小</h5>
     <h6>最小，蚂蚁级别</h6>
     ```
   - 效果：`<h1>` 字最大，`<h2>` 次之，依次变小。标题自带加粗和换行。
   - **坑提醒**：
     - 别乱用 `<h1>`，一个网页最好只有一个 `<h1>`，表示主标题，SEO（搜索引擎优化）会更友好。
     - 标签没闭合（比如 `<h1>xxx`），浏览器可能自动补全，但排版可能乱。
     - 不要跳级用（比如直接从 `<h1>` 到 `<h3>`），保持层级清晰。

2. **段落标签：`<p>`**
   - 用来写一段文字，自动换行，段落间有空隙。
   - 示例：
     ```html
     <p>这是一段话，写点啥都行，爽就完事儿了。</p>
     <p>这是另一段话，自动换行，牛逼。</p>
     ```
   - 效果：两段文字，中间有空行。
   - **坑提醒**：
     - `<p>` 里不能直接放其他块级标签（比如 `<h1>`），否则浏览器可能报错或渲染乱。
     - 想在段落里换行，用 `<br>`，别用多个 `<p>`。

3. **链接标签：`<a>`**
   - 用来加超链接，点一下跳转到别的网页。
   - 格式：`<a href="网址">链接文字</a>`
   - 示例：
     ```html
     <a href="https://www.bilibili.com">点我去 B 站</a>
     <a href="https://www.douyin.com">点我去抖音</a>
     ```
   - 效果：显示“点我去 B 站”，点一下跳转到 B 站。
   - **坑提醒**：
     - `href` 忘了写或写错（比如漏了 `https://`），链接点不动。
     - 想新窗口打开链接，加 `target="_blank"`，比如 `<a href="https://www.bilibili.com" target="_blank">去 B 站</a>`。
     - 链接文字别写太长，简短点用户体验好。

4. **图片标签：`<img>`**
   - 插入图片，单身狗标签，不需要闭合。
   - 格式：`<img src="图片路径" alt="图片描述">`
   - 示例：
     ```html
     <img src="cool_cat.jpg" alt="一只酷炫的猫">
     <img src="https://example.com/dog.png" alt="一只狗">
     ```
   - 说明：
     - `src`：图片路径，可以是本地文件（比如 `cool_cat.jpg`），也可以是网上 URL。
     - `alt`：图片描述，图片加载失败时显示这文字，也方便盲人用户。
   - **坑提醒**：
     - 图片路径错误（文件名大小写不对、文件不在文件夹里），图片显示为叉叉或 `alt` 文字。
     - 本地图片路径用相对路径（比如 `cool_cat.jpg`），别用绝对路径（比如 `C:\Users\XXX\cool_cat.jpg`），否则换电脑就失效。
     - 图片太大（比如 10MB），加载慢，建议压缩到 1MB 以下。
     - 忘了写 `alt`，SEO 和无障碍体验会差。

5. **换行标签：`<br>`**
   - 单身狗标签，强制换行。
   - 示例：
     ```html
     <p>第一行<br>第二行</p>
     ```
   - 效果：第一行和第二行中间换行，但还在一个段落里。
   - **坑提醒**：
     - 别用一堆 `<br>` 控制间距，后面学 CSS 再搞样式。
     - `<br>` 写成 `<br />` 也行，但 HTML5 不强求。

6. **注释：`<!-- -->`**
   - 写代码时的备注，浏览器不显示。
   - 示例：
     ```html
     <!-- 这是一个标题，下面是段落 -->
     <h1>标题</h1>
     <p>段落</p>
     ```
   - **坑提醒**：
     - 注释别嵌套（比如 `<!-- xxx <!-- yyy --> -->`），会报错。
     - 别把注释写在标签中间（比如 `<h1><!-- xxx -->标题</h1>`），可能乱。

## 实战：写一个完整的网页

```html

<html lang="zh-CN">

    <meta charset="UTF-8">
    <title>我的牛逼网页</title>
</head>
<body>
    <!-- 主标题 -->
    <h1>欢迎体验我的第一个网页，操！</h1>
    
    <!-- 关于我 -->

    <p>我是一个刚学 HTML 的小白，今天是第一天，感觉他妈的爽爆了！</p>

    <p>我喜欢看 B 站和抖音，点下面链接去嗨：</p>
    <a href="https://www.bilibili.com" target="_blank">去 B 站看视频</a><br>
```

## 常见问题及解决：别他妈抓狂

小白学 HTML 总会踩坑，下面是常见问题和解决办法，记好了：

1. **网页乱码，中文全是方块？**
   - **原因**：没写 `<meta charset="UTF-8">`，或文件编码不对。
   - **解决**：
     - 确认 `<head>` 里有 `<meta charset="UTF-8">`。
     - VS Code 保存文件时，底部状态栏确认编码是 UTF-8（不是 GBK 或其他）。
     - 记事本保存时，选“UTF-8”编码。

2. **图片显示不出来？**
   - **原因**：图片路径错误、文件不存在、格式不支持。
   - **解决**：
     - 检查 `src` 路径，文件名大小写一致（`Cool_Cat.jpg` 和 `cool_cat.jpg` 不一样）。
     - 确认图片在文件夹里，或用网上图片 URL。
     - 用 jpg/png 格式，gif 也行，别的格式可能不兼容。
     - 路径用 `./cool_cat.jpg`（当前文件夹）或 `cool_cat.jpg`，别用绝对路径。

3. **链接点不动？**
   - **原因**：`href` 写错或漏了。
   - **解决**：
     - 确认 `href` 是完整 URL（比如 `https://www.bilibili.com`）。
     - 检查有没有拼写错误（比如 `http` 写成 `htp`）。
     - 想新窗口打开，加 `target="_blank"`。

4. **网页啥都没显示？**
   - **原因**：标签没写对、文件没保存、路径错误。
   - **解决**：
     - 检查 `<body>` 里有没有内容，标签是否闭合。
     - 确认文件保存为 `.html` 后缀。
     - 用 Live Server 或拖到浏览器，确保打开正确文件。

5. **排版乱七八糟？**
   - **原因**：标签嵌套错误、漏闭合、浏览器缓存。
   - **解决**：
     - 检查标签嵌套，比如 `<h1><p>xxx</p></h1>` 是错的，改成 `<h1>xxx</h1><p>yyy</p>`。
     - 确认所有标签闭合（比如 `<p>xxx</p>`，别漏 `</p>`）。
     - 刷新浏览器，按 **Ctrl + F5** 清缓存。

6. **VS Code 或 Live Server 抽风？**
   - **原因**：插件没装好、端口占用、文件没保存。
   - **解决**：
     - 确认 Live Server 插件已安装并启用（VS Code 底部有“Go Live”按钮）。
     - 端口被占（比如 5500），在 VS Code 设置里改端口。
     - 每次改代码后保存（Ctrl + S），Live Server 才会刷新。

7. **代码没效果，浏览器报错？**
   - **原因**：语法错误、浏览器不兼容。
   - **解决**：
     - 打开 Chrome 开发者工具（F12），看“Console”标签有没有错误提示。
     - 比如提示“Unexpected token”，检查标签是否写错（比如 `<h1>xxx<h1>`）。
     - 用最新版 Chrome，避开老浏览器兼容问题。

3. **加链接**：
   - 加一个指向你喜欢的网站链接（比如微博、淘宝）。
   - 示例：`<a href="https://www.weibo.com" target="_blank">去微博浪一浪</a>`。
   - **坑提醒**：确认 URL 正确，测试链接能不能点。

4. **加图片**：
   - 再加一张图片，写个描述（比如“这是我家狗，帅炸了”）。
   - 示例：`<img src="my_dog.jpg" alt="一只帅气的狗">`。
   - **坑提醒**：图片放文件夹里，路径写对。

5. **加注释**：
   - 在每个 `<h2>` 前加注释，说明这部分内容（比如 `<!-- 我的爱好部分 -->`）。
   - **坑提醒**：注释别写在标签中间，保持代码整洁。

**练习目标**：改完代码，网页应该有：
- 新标题和主标题。
- 至少 3 段文字（关于我、爱好、计划）。
- 至少 2 个链接（B 站 + 你加的）。
- 至少 2 张图片（猫 + 你加的）。
- 注释清晰，代码看着舒服。

保存，刷新，看效果。爽不爽？

## 进阶小技巧：第一天多学点

虽然是第一天，但你这么牛逼，给你点额外技巧，装逼更带劲：

1. **控制图片大小**：
   - 用 `width` 和 `height` 属性调整图片大小。
   - 示例：`<img src="cool_cat.jpg" alt="猫" width="200" height="150">`（单位是像素）。
   - **坑提醒**：
     - 只设 `width`，`height` 会自动按比例缩放，推荐这样用。
     - 别设太大，图片会变形或加载慢。

2. **加空行或空格**：
   - HTML 默认忽略多余空格和回车，想加空格用 `&nbsp;`，想空行用 `<br>`。
   - 示例：
     ```html
     <p>第一行<br><br>第三行</p>
     <p>单词&nbsp;&nbsp;间隔</p>
     ```
   - **坑提醒**：别用一堆 `<br>` 控制布局，CSS 才是干这个的。

3. **用 `<div>` 组织内容**：
   - `<div>` 是个容器标签，啥都能装，用来分组内容。
   - 示例：
     ```html
     <div>
         <h2>我的爱好</h2>
         <p>我喜欢看 B 站。</p>
     </div>
     <div>
         <h2>我的图片</h2>
         <img src="cool_cat.jpg" alt="猫">
     </div>
     ```
   - **坑提醒**：`<div>` 本身没样式，后面学 CSS 再让它好看。

4. **检查代码规范**：
   - 用 VS Code 的“格式化文档”（右键 > 格式化文档），让代码缩进整齐。
   - 装个 **Prettier** 插件，自动格式化代码，爽。
   - **坑提醒**：格式化前保存文件，不然可能丢代码。
